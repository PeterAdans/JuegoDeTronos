Formularios HTML

Los formularios en HTML, son elementos que nos van a dotar de cierta funcionalidad t dinamismo a nuestra página web.
Nos van a permitir interactuar con nuestras páginas web y por lo tanto, son elementos fundamentales para poder recoger información
para posteriormente gestionarla y almacenarla con el fin que queramos.
Los formularios nos permiten dos tipos de funciones principales:
  • Recoger información del usuario
  • Enviar esos datos que se han guardado, para procesarlos o realizar cualquier tipo de operación que se requiere con ellos

Los principales objetivos que tenemos que tener en cuenta a la hora de diseñar un formulario en HTML, son los siguientes:
  • El formulario tiene que ser lo más sencillo posible a la hora de insertar datos por parte del usuario
  • La experiencia del usuario / usabilidad (UX) tiene que ser lo más agradable posible
  • Los datos introducidos en los formularios tienen que tener un formato predecible y esperado. Por ejemplo: si vamos a introducir
    el nombre del usuario, facilitamos que el usuario sepa que tiene que introducir en dicho apartado su nombre...
  • Reducir los errores al introducir los datos en la medida de lo posible y comunicar claramente al usuario si se está
    dando este problema

Etiquetas básicas para la creación de un formulario en HTML

Para crear un formulario en HTML, utilizaremos la etiqueta form con apertura y cierre.
A partir de utilizar la etiqueta form que establece que vamos a crear (usar) un formulario en HTML, tenemos que tener en cuenta una
serie de propiedades que vamos a usar con ella:
  • action: Indica la URL o BBDD o web o correo... (dirección Backend) a la que queremos enviar la información de nuestro
    formulario.
  • method: Método puede ser get o post.Get se utiliza cuando queremos mostrar la información que se envia
    de nuestro formulario a traves del protocolo HTTP/HTTPS de nuestra web. Post se utiliza cuando queremos ocultar la información
    de nuestro formulario a traves del protocolo HTTP/HTTPS.
  • name:se utiliza para dar un nombre al formulario
  • target: se utiliza para abrir el formulario en una pestaña nueva (target=_blank)
  • entype: codificar la información del envio del formulario. Es lo que se podría llamar un token.
  • autocomplete: Tiene una función on u off, que sirve para activar o desactivar el autocompletado de
    nuestro formulario en la web.

Tipos de datos que podemos gestionar en un formulario HTML

En un formulario, podemos gestionar cualquier tipo de información que se quiera/requiera. Pero los más utilizados son los siguientes:
  • Información de texto: Nombres, apellidos, direcciones físicas, etc. Utilizaremos las etiquetas input y textarea.
    (input type = text)
  • Numeros o cantidades: Edades, precios, cuantias, etc. Utilizaremos elementos input pero del tipo number (input type=number)
  • Fechas u horas: fechas de nacimiento, calendario, etc. Utilizaremos el input type date, input type = week, etc.
  • Opciones de verdadero o falso: Utilizaremos el tipo input type = checkbox.
  • Opciones únicas:podremos elegir entre 2 o + opciones. type = radio o select.
  • Opciones únicas simples: donde utilizaremos select múltiple, input checkbox, etc.
  • Opciones abiertas: datalist
  • Seleccionar colores: input type=color
  • Seleccionar archivos: input type = file

Campos de entrada de información numericos

Para poder obtener informacion numerica de nuestros formularios, podemos utilizar dos tipos de campos existentes:
  • input type=number con el cual estamos indicando que el valor de entrada será un número
  • Campo de rango: input type=range
  • Los atributos que van asociados a estos son: min (establece el valor minimo para pasar la validación),
    max(numero maximo parmitido para pasar la validacion) el step (permite la variacion entre numeros para pasar la validacion)
